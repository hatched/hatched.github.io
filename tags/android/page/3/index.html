When to update your semver version number

April 25, 2014

3 minute read

Anyone who is familiar with package versioning has used, or at the very least heard of, Semantic Versioning. For the uninitiated, semver is a three part version number in the format MAJOR.MINOR.PATCH ex) 1.13.2, and you can find very in-depth details on the semver website. The semver website outlines the rules for incrementing version numbers as: MAJOR: When you make incompatible API changes. MINOR: When you add functionality in a backwards-compatible manner.
